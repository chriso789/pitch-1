

# Inspection Report PDF Generator

## Overview

Add a "Download Report" button to completed inspections that generates a professional, branded PDF with all photos, notes, property details, and inspection metadata. The PDF will be built using the existing `jsPDF` library (already installed), constructing pages programmatically rather than relying on html2canvas -- this ensures consistent output regardless of screen state and allows embedding images directly.

## New Files

### 1. `src/components/inspection/useInspectionReportPDF.ts`

A custom hook that generates the PDF programmatically using `jsPDF`:

**Core logic:**
- Accept inspection data (steps_data, property address, inspector name, date, company info)
- Page 1: Cover/header with company branding, property details, inspection summary stats
- Subsequent pages: Each step rendered as a section with:
  - Step number and title (bold)
  - Photo embedded as JPEG (fetched via fetch() then converted to base64 for jsPDF.addImage)
  - Notes text wrapped below
  - "Skipped" badge if no photo was taken
- Footer on every page: page number, "Generated by PITCH CRM", timestamp
- Returns a Blob for download or upload to Supabase Storage

**Key implementation details:**
- Use `jsPDF` directly (no html2canvas needed -- avoids CORS issues with storage photos)
- Fetch each photo URL, convert to base64 using canvas, embed with `pdf.addImage()`
- Handle page breaks automatically: track Y position, add new page when content exceeds page height
- Use Inter font family via jsPDF's built-in Helvetica as fallback (jsPDF doesn't support custom web fonts natively without plugins, but Helvetica is professional enough for reports)

### 2. `src/components/inspection/InspectionReportPreview.tsx`

A dialog component that:
- Fetches inspection data from the `inspections` table by ID
- Shows a loading state while generating
- Offers "Download PDF" and "Share" buttons
- Optionally uploads the PDF to Supabase Storage for sharing

## Technical Details

### Photo Embedding Strategy

Since photos are in Supabase Storage (public URLs), we fetch them as blobs, convert to base64 data URLs, then embed via `jsPDF.addImage()`. This avoids CORS issues that html2canvas would face.

```typescript
async function fetchImageAsBase64(url: string): Promise<string> {
  const response = await fetch(url);
  const blob = await response.blob();
  return new Promise((resolve) => {
    const reader = new FileReader();
    reader.onloadend = () => resolve(reader.result as string);
    reader.readAsDataURL(blob);
  });
}
```

### Page Layout Constants

- Page: Letter size (215.9mm x 279.4mm)
- Margins: 15mm all sides
- Photo size: 80mm x 60mm per step
- Font sizes: Title 18pt, Step headers 12pt, Body 10pt, Footer 8pt
- Max 2-3 steps per page depending on notes length

### Inspection History Query

```typescript
const { data: inspections } = await supabase
  .from('inspections')
  .select('*')
  .eq('lead_id', leadId)
  .order('created_at', { ascending: false });
```
