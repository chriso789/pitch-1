

# Fix: Extend Setup Link Validity to 4+ Hours

## Root Cause

Supabase's `generateLink` for `recovery` type creates OTP tokens that expire based on the project's Auth settings. The default is **3600 seconds (1 hour)**. Your edge functions (`create-company-user`, `resend-user-invitation`, `admin-create-user`, `provision-tenant-owner`, etc.) all use `generateLink({ type: 'recovery' })` which inherits this server-side expiry.

The app code cannot override this — it's a Supabase Auth project setting.

## Fix — Two Parts

### Part 1: Supabase Dashboard Setting (Manual — User Action Required)

Go to the Supabase Dashboard:
**Authentication → URL Configuration → Email OTP Expiration**

Change the value from `3600` (1 hour) to `14400` (4 hours) or `86400` (24 hours for maximum flexibility).

Direct link: https://supabase.com/dashboard/project/alxelfrbjzkmtnsulcei/auth/url-configuration

This single change affects ALL recovery/invite token lifetimes generated by `generateLink`.

### Part 2: Code Fix — Improve Error Message & Add Retry UX

Update `SetupAccount.tsx` to show a clearer error when tokens expire, and make "Request New Link" actually work by calling the `resend-user-invitation` edge function or redirecting to a page where the user can enter their email.

Currently `handleRequestNewLink` navigates to `/request-setup-link` which may not exist. Update it to navigate to `/login` with a toast message, or better yet, add an inline "resend" flow directly on the error screen.

### Files Changed
- `src/pages/SetupAccount.tsx` — improve expired token error UX with inline resend capability
- **Supabase Dashboard** — increase OTP expiration to 14400s (user must do manually)
